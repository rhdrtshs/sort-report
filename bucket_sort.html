1. 演算法原理

桶排序會先根據數值範圍把資料分散到多個桶（bucket）中，再對每個桶內元素排序，最後依桶的順序串接回原陣列。

特性：若資料分布均勻，平均時間可接近線性；效能受資料分布與桶內排序方法影響。

2. 偽代碼 

BUCKET-SORT(A)
  create k empty buckets
  for each x in A
      put x into bucket[index(x)]
  for i = 0 to k-1
      sort bucket[i]
  concatenate bucket[0], bucket[1], ... , bucket[k-1]

C 語言參考實作 

#include <stdio.h>

#define BUCKET_COUNT 10
#define MAX_PER_BUCKET 100

void insertion_sort(int arr[], int n) {
    for (int i = 1; i < n; i++) {
        int key = arr[i];
        int j = i - 1;
        while (j >= 0 && arr[j] > key) {
            arr[j + 1] = arr[j];
            j--;
        }
        arr[j + 1] = key;
    }
}

void bucket_sort(int arr[], int n) {
    int buckets[BUCKET_COUNT][MAX_PER_BUCKET] = {0};
    int bucket_sizes[BUCKET_COUNT] = {0};

    // 範例假設資料介於 0~99，依十位數分桶
    for (int i = 0; i < n; i++) {
        int idx = arr[i] / 10;
        if (idx < 0) idx = 0;
        if (idx >= BUCKET_COUNT) idx = BUCKET_COUNT - 1;
        buckets[idx][bucket_sizes[idx]++] = arr[i];
    }

    for (int i = 0; i < BUCKET_COUNT; i++) {
        insertion_sort(buckets[i], bucket_sizes[i]);
    }

    int write = 0;
    for (int i = 0; i < BUCKET_COUNT; i++) {
        for (int j = 0; j < bucket_sizes[i]; j++) {
            arr[write++] = buckets[i][j];
        }
    }
}

int main(void) {
    int arr[] = {29, 25, 3, 49, 9, 37, 21, 43};
    int n = sizeof(arr) / sizeof(arr[0]);

    bucket_sort(arr, n);

    for (int i = 0; i < n; i++) {
        printf("%d ", arr[i]);
    }
    printf("\n");
    return 0;
}

3. 複雜度

項目結果說明
平均時間O(n + k)資料分布均勻時，分桶與回收近似線性，桶內排序負擔小。
最壞時間O(n^2)若大量元素集中在少數桶，桶內排序會退化為二次方等級。
最佳時間O(n + k)元素分散且桶內接近有序時，桶內排序成本很低。
空間複雜度O(n + k)需要額外桶結構與桶內儲存空間。
穩定性可穩定（視實作）若分桶與桶內排序皆保留相等元素順序，整體可維持穩定。
